Define uniform ground planes

Load a new FastHenry2 input file

  1. Select the FastHenry2 View (the view should be in foreground with the caption highlighted).
  2. From the File menu select File->Close FastHenry2 File.
  3. From the File menu select File->Open FastHenry2 File or click on the icon on the toolbar.
  4. Browse to the software installation directory (e.g. C:\Program Files\FastFieldSolvers) and under FastHenry2\Samples\Input_files select the file simple_gp.inp, then click Open.

    A model of a plane with a L-shaped line above it will appear in the FastModel View.

Examine the input file

We will now focus on the new aspects of this file.

  1. Examine the ground plane definition.

    *  First define 3 corners of the plane, in clockwise or counter-cw order
    g1 x1 = 0     y1 = 0     z1 = 0
    +  x2 = 1000  y2 = 0     z2 = 0
    +  x3 = 1000  y3 = 1000  z3 = 0
    *     thickness:
    +  thick = 1.2
    *     discretization:
    +  seg1 = 20 seg2 = 20
    *     nodes to reference later:
    +  nin  (800,800,0)
    +  nout (0,200,0)

    First of all note that a line beginning with the '+' character is a continuation of the previous line.

    The ground plane is defined by three of its four corners, in our case (0,0,0), (1000,0,0) and (1000,1000,0). The default units (see the .Units directive at the beginning of the file) are mils, so these corners specify a 1000 mil x 1000 mil plane. The plane is also 1.2 mil thick.

    To finely represent the current flow and distribution on this plane, the plane is discretized into 20 x 20 segments. This means that the ground plane definition is in fact only a way to instruct FastHenry2 on how to lay out nodes and segments to build a discretized plane. You can see the effect of this discretization directly in FastModel; you can also change the discretization and see how this reflects on the plane in the FastModel View.

    So FastHenry2 has laid out a mesh of nodes for you to construct the ground plane. You must have a way to reference any of these nodes to be able to preform electrial connections. You can therefore specify explicitly any number of nodes (in our example only two, named nin and nout) and they will refer to the internal nodes of the plane which are closest to the supplied coordinates.

  2. Examine the .equiv statement

    .equiv nin n3

    The .equiv keyword shorts two nodes together. In this case, the .equiv is used to connect the L-shaped conductor with the ground plane.

    Note that segments geometrically touching are NOT considered electrically connected. Segments are electrically connected only if they share a node.

  3. Launch a FastHenry2 simulation. You get the loop inductance of the trace.
